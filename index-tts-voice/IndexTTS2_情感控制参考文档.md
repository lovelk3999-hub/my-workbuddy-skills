## 一、Index-TTS2 简介

IndexTTS2 是 B站 开源的情感表达与音色解耦的零样本语音合成模型。
核心特性: **情感表达与说话人身份解耦，可独立控制音色和情感**。

官方仓库: https://github.com/index-tts/index-tts

## 二、全部情感控制参数

### 2.1 infer() 方法签名

```python
def infer(
    spk_audio_prompt,     # 音色参考音频路径 (必填)
    text,                 # 合成文本 (必填)
    output_path,          # 输出路径 (必填)
    # === 情感控制参数 ===
    emo_audio_prompt=None, # 情感参考音频(另一段带情绪的录音)
    emo_alpha=1.0,        # 情感强度 0.0~1.0 (默认满值)
    emo_vector=None,       # 8维情感向量 [高,愤,悲,怕,厌,忧,惊,平]
    use_emo_text=False,    # 是否用文本自动推断情感(QwenEmotion)
    emo_text=None,         # 情感描述文本(可不同于合成文本)
    use_random=False,      # 随机情感采样 ⚠️ 会降低音色
    # === 其他参数 ===
    interval_silence=200,  # 多段合成时段间静音(ms)
    max_text_tokens_per_segment=120,  # 每段最大token数
    verbose=False,         # 是否打印详细信息
    **generation_kwargs    # 其他生成参数
)
```

### 2.2 4种情感控制模式

| 模式 | 方式 | 参数配置 | 适用场景 |
|:---:|:----|:--------|:--------|
| 0 | 与参考音频相同 | 默认(不传情感参数) | 纯音色克隆，最保真 |
| 1 | 情感参考音频 | `emo_audio_prompt="xxx.wav"` | 有现成情感录音 |
| 2 | **情感向量 ⭐** | `emo_vector=[...]`, `emo_alpha=0.4` | **精准控制，本项目用这个** |
| 3 | 情感描述文本 | `use_emo_text=True`, `emo_text="极度悲伤"` | 不想手动调向量时 |

### 2.3 8维情感向量详解

**顺序固定，不可调换:**

```
[高兴, 愤怒, 悲伤, 害怕, 厌恶, 忧郁, 惊讶, 平静]
```

**内置情感偏置表**（防止某些情感出怪声）:

| 维度 | 英文 | 偏置系数 | 说明 |
|:----|:----|:--------:|:----|
| 高兴 | happy | 0.9375 | 几乎不压制 |
| 愤怒 | angry | 0.875 | 稍有压制 |
| 悲伤 | sad | 1.0 | 不压制 |
| 害怕 | afraid | 1.0 | 不压制 |
| 厌恶 | disgusted | 0.9375 | 几乎不压制 |
| 忧郁 | melancholic | 0.9375 | 几乎不压制 |
| **惊讶** | **surprised** | **0.6875** | **压制最多（易出怪声）** |
| **平静** | **calm** | **0.5625** | **最安全** |

**处理流程:**

```
用户传入向量 → 乘以偏置 → 总和超0.8则等比压缩 → 乘以emo_alpha → 送模型
```

### 2.4 emo_alpha 情感强度 0.0~1.0

| alpha | 效果 | 音色影响 |
|:----:|:----|:--------:|
| 0.0 | 纯音色克隆,情感归零 | ✅ 完全保真 |
| 0.2 | 极轻微,几乎无感 | ✅ 保真 |
| **0.4** | **轻微可感知,本项目选中值** | **✅ 保真(本项目确认)** |
| 0.6 | 中等,效果明显 | ⚠️ 可能开始偏移 |
| 0.8 | 较强 | ❌ 音色明显改变 |
| 1.0 | 满情感(默认) | ❌ 音色变化最大 |

```
录音能量分布:

 10- 20s | ██████
 20- 30s | ███████
```

**关键发现**: Index-TTS 自动截取参考音频前 330750 采样(≈13.8秒)。
如果开头质量差,音色克隆效果会差。建议**手动裁剪最优15秒片段**作为参考。

最终参考: `语音\参考音色_最佳15s.wav` (60-75秒, 24000Hz mono)

### 3.3 第一次合成测试

| 项目 | 值 |
|:----|:----|
| 文本 | "聊起AI提效，你是不是觉得，AI帮你干活离你太远？" |

### 3.4 情感边界测试(惊讶)

用 `use_emo_text=True` (QwenEmotion自动检测):

```
检测结果: {高兴:0, 愤怒:0.1, 悲伤:0.1, 害怕:0.05, 厌恶:0.1, 忧郁:0.1, 惊讶:0.4, 平静:0.15}
```

5梯度对比: alpha=0.0, 0.2, 0.4, 0.6, 0.8

| alpha | 文件名 | 结果 |
|:----:|:-------|:----|
| 0.0 | 测试_边界_alpha_00_纯音色.wav | 纯克隆 |
| 0.2 | 测试_边界_alpha_02_极轻微.wav | 几乎无感 |
| **0.4** | 测试_边界_alpha_04_轻微.wav | **✅ 确认边界(音色保留+情感可感知)** |
| 0.6 | 测试_边界_alpha_06_中等.wav | 音色开始偏移 |
| 0.8 | 测试_边界_alpha_08_较强.wav | 音色明显变化 |

**结论: 惊讶维度在 alpha=0.4 是最佳平衡点**

### 3.5 情感边界测试(高兴/得意)

用自定义向量: `[高兴0.7, 平静0.3]`，alpha=0.4 vs 0.8

| alpha | 文件名 | 结果 |
|:----:|:-------|:----|
| **0.4** | 测试_高兴alpha04.wav | **✅ 音色还行(确认)** |
| 0.8 | 测试_高兴alpha08.wav | 极端对比用 |

**结论: 高兴维度同样适用 alpha=0.4 边界。但注意高兴偏置高(0.9375)，同样alpha下高兴效价比惊讶高约70%**

## 六、注意事项

1. **参考音频不要用全长录音** - 模型只取前14秒,应手动裁剪最优15秒片段
2. **emo_vector 和 use_emo_text 互斥** - 同时传时 use_emo_text 优先
3. **emo_alpha 对高兴更敏感** - 高兴偏置0.9375 vs 惊讶偏置0.6875，同样alpha下高兴效价比惊讶高约70%
4. **use_random 会降低音色** - 官方明确说"开启随机采样会降低音色的还原度"
5. **首次运行慢** - 需要下载 w2v-bert-2.0(~2.16GB)等辅助模型,后续缓存后快很多
6. **BigVGAN CUDA kernel 报错可忽略** - 回退到 torch 实现,不影响结果
